                    TICKET GENERATOR - FIRST STEPS
================================================================================

Hi! Welcome to your Ticket Generator application.

This file will guide you through the ABSOLUTE BASICS to get started.

================================================================================
                        WHAT YOU NEED FIRST
================================================================================

1. Node.js installed (download from: https://nodejs.org)
2. A Gmail account
3. That's it!

================================================================================
                    STEP 1: INSTALL PACKAGES (2 minutes)
================================================================================

Open Command Prompt or Terminal in this folder and type:

    npm install

Then:

    cd frontend
    npm install
    cd ..

Wait for it to finish. You'll see lots of text - that's normal!

================================================================================
                    STEP 2: CREATE YOUR .env FILE (3 minutes)
================================================================================

1. Create a new file in this folder called:   .env

2. Open it in Notepad and paste this:

MONGO_URI=mongodb://localhost:27017/ticket-generator
EMAIL=your-email@gmail.com
EMAIL_PASSWORD=xxxx xxxx xxxx xxxx
PORT=5000

3. Replace "your-email@gmail.com" with YOUR Gmail address

4. For EMAIL_PASSWORD, you need a special password:
   - Go to: https://myaccount.google.com/security
   - Turn on "2-Step Verification" if not already on
   - Go to: https://myaccount.google.com/apppasswords
   - Create a new App Password
   - Copy the 16-character password (looks like: abcd efgh ijkl mnop)
   - Paste it after EMAIL_PASSWORD=

5. For MONGO_URI, you can either:
   - Keep it as is (if you have MongoDB installed locally)
   - OR use free cloud MongoDB:
     * Go to: https://cloud.mongodb.com
     * Sign up (free)
     * Create a cluster (free M0 tier)
     * Get connection string and replace the MONGO_URI line

6. SAVE the .env file

For detailed help, see: env.config.txt

================================================================================
                    STEP 3: CHECK IF IT'S READY (30 seconds)
================================================================================

In Command Prompt/Terminal, type:

    npm run verify

You should see green checkmarks (âœ“). If you see red X's, something is wrong.

================================================================================
                    STEP 4: START THE APPLICATION (1 minute)
================================================================================

OPTION A - SUPER EASY (Windows only):
    Just double-click the file:  start-dev.bat
    
    Two windows will open and your browser will open automatically!

OPTION B - MANUAL (All systems):
    Open TWO Command Prompt/Terminal windows:
    
    Window 1 - type:
        npm run dev
    
    Window 2 - type:
        cd frontend
        npm run dev
    
    Then open your browser to: http://localhost:5173

================================================================================
                    STEP 5: TEST IT! (2 minutes)
================================================================================

1. You should see a black page with a white form

2. Fill in the form:
   - Name: Test User
   - Registration Number: TEST001
   - Batch: 2021-2025
   - Email: your-email@gmail.com

3. Click "Generate Ticket"

4. Check your email - you should receive a PDF ticket!

================================================================================
                        COMMON PROBLEMS
================================================================================

PROBLEM: "npm: command not found"
SOLUTION: You need to install Node.js from https://nodejs.org

PROBLEM: "MongoDB connection error"
SOLUTION: Either install MongoDB locally OR use MongoDB Atlas (cloud)
          See env.config.txt for MongoDB Atlas setup

PROBLEM: "Email authentication failed"
SOLUTION: Make sure you're using App Password, not regular Gmail password
          See Step 2 above for how to get App Password

PROBLEM: "Module not found"
SOLUTION: Run: npm install
          Then: cd frontend && npm install && cd ..

PROBLEM: ".env file not found"
SOLUTION: Create .env file in the main folder (not in frontend folder)
          Follow Step 2 above
